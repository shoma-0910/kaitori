## 1. ログイン画面

### 画面の説明
アプリを開くと最初に表示される画面です。

### 画面の項目

| 項目名 | 説明 |
|--------|------|
| メールアドレス入力欄 | 登録済みのメールアドレスを入力 |
| パスワード入力欄 | パスワードを入力（非表示） |
| 「ログイン」ボタン | クリックしてログイン |
| 「新規登録」タブ | 新しくアカウントを作る場合にクリック |

### 操作の流れ

**ログインする場合：**
```
① メールアドレスを入力
    ↓
② パスワードを入力
    ↓
③ 「ログイン」ボタンをクリック
    ↓
④ ダッシュボードに移動
```

**新規登録する場合：**
```
① 「新規登録」タブをクリック
    ↓
② メールアドレスを入力
    ↓
③ パスワードを入力
    ↓
④ 名前を入力
    ↓
⑤ 「登録」ボタンをクリック
    ↓
⑥ ダッシュボードに移動
```

## 3. ダッシュボード

### 画面の説明
システム全体の状況を一目で確認できる画面です。

### 画面の構成

#### ヘッダー部分
| 項目 | 説明 |
|------|------|
| タイトル「ダッシュボード」 | ページ名 |
| 組織バッジ | 所属している組織名が表示される |

#### KPIカード（4つ）

| カード名 | 説明 | 例 |
|----------|------|-----|
| 対象店舗数 | 登録されている店舗の合計数 | 12 |
| 予定催事件数 | これから開催予定のイベント数 | 5 |
| 総実績粗利 | これまでの利益の合計 | ¥1,500,000 |
| 総概算コスト | これまでのコストの合計 | ¥300,000 |

#### グラフエリア

| グラフ名 | 説明 |
|----------|------|
| 店舗別分析（棒グラフ） | 各店舗の粗利・コストを比較 |
| 催事別粗利分布（円グラフ） | 各催事の粗利割合を表示 |
| 店舗別粗利分布（円グラフ） | 各店舗の粗利割合を表示 |
| 店舗別粗利ランキング | 粗利の高い店舗順に表示 |
| 月別粗利推移 | 月ごとの粗利の変化を表示 |

### 操作の流れ
```
ダッシュボードを開く
    ↓
KPIカードで全体状況を確認
    ↓
グラフで詳細な傾向を分析
```

## 4. 店舗選定

### 画面の説明
新しい催事場所（スーパーマーケット）を探して登録する画面です。
※予約リクエストは「登録店舗」ページから行います。

### 画面の構成

#### 地域情報検索カード
| 項目 | 説明 |
|------|------|
| 地域名入力欄 | 検索したい地域を入力（例：渋谷区） |
| 「検索」ボタン | 地域情報を取得 |

#### 地域情報表示エリア（検索後に表示）
| 項目 | 説明 |
|------|------|
| 人口 | その地域の総人口 |
| 平均年齢 | 住民の平均年齢 |
| 平均年収 | 住民の平均年収 |
| 外国人比率 | 外国人住民の割合 |
| 男女比 | 男性・女性の割合 |
| 年齢分布 | 年代別の人口割合 |
| 「AI推定」バッジ | データがAI推定の場合に表示 |

#### マップエリア
| 項目 | 説明 |
|------|------|
| Googleマップ | 地域の地図を表示 |
| マーカー（ピン） | スーパーマーケットの位置 |
| 「このエリアで検索」ボタン | 表示エリア内の店舗を取得 |

### ポップアップ（店舗詳細モーダル）

マーカーをクリックすると表示されるポップアップです。

| 項目 | 説明 |
|------|------|
| 店舗名 | スーパーの名前 |
| 住所 | 所在地 |
| 電話番号 | 連絡先（クリックで電話） |
| 営業時間 | 曜日ごとの営業時間 |
| ウェブサイト | 公式サイトへのリンク |
| 「この店舗を登録」ボタン | 店舗を登録店舗に追加 |

### 操作の流れ

**店舗を探す：**
```
① 地域名を入力（例：新宿区）
    ↓
② 「検索」ボタンをクリック
    ↓
③ 地域の人口統計情報が表示される
    ↓
④ マップで地図を移動
    ↓
⑤ 「このエリアで検索」をクリック
    ↓
⑥ 周辺のスーパーがマーカーで表示される
```

**店舗を登録する：**
```
① マーカーをクリック
    ↓
② 店舗詳細モーダルが開く
    ↓
③ 「この店舗を登録」ボタンをクリック
    ↓
④ 「登録しました」と表示される
    ↓
⑤ 登録店舗一覧に追加される
    ↓
⑥ 予約リクエストは「登録店舗」ページから行う
```

## 5. 登録店舗

### 画面の説明
登録済みのスーパーマーケットを管理する画面です。
**予約リクエストはこのページから行います。**

### 画面の構成

#### フィルターエリア
| 項目 | 説明 |
|------|------|
| 都道府県で絞り込み | 都道府県を選択して絞り込み |
| 店舗数表示 | 全X件中Y件表示と表示 |

#### 店舗カード（各店舗ごと）
| 項目 | 説明 |
|------|------|
| 店舗名 | スーパーの名前 |
| 住所 | 所在地 |
| 電話番号 | 連絡先 |
| ランクバッジ | S/A/B/Cのランク表示 |
| 粗利情報 | 総粗利・最新粗利日 |
| 「粗利を追加」ボタン | 売上を入力 |
| 「詳細」ボタン | 詳細モーダルを開く |
| 「ゴミ箱」アイコン | 店舗を削除 |

### ポップアップ

#### 店舗詳細モーダル

**「基本情報」タブ**
| 項目 | 説明 |
|------|------|
| 住所 | 所在地 |
| 電話番号 | クリックで電話 |
| ウェブサイト | 公式サイトへのリンク |
| 営業時間 | 曜日ごとの営業時間 |
| ランク | S/A/B/Cのランク |

**「予約」タブ**
| 項目 | 説明 |
|------|------|
| 開始日 | 催事の開始日 |
| 終了日 | 催事の終了日 |
| 担当者名 | 担当者を入力 |
| 「予約リクエストを送信」ボタン | 予約を申請 |

**「売上履歴」タブ**
| 項目 | 説明 |
|------|------|
| 売上一覧 | 過去の売上データを表示 |
| 「売上を追加」ボタン | 新しい売上を入力 |

**「周辺施設」タブ**
| 項目 | 説明 |
|------|------|
| 周辺施設一覧 | 半径100m以内の施設を表示 |

#### 粗利追加ダイアログ
| 項目 | 説明 |
|------|------|
| 日付 | 売上の日付を選択 |
| 粗利金額 | 金額を入力（円） |
| 買取品目数 | 買い取った品目の数 |
| メモ | 補足情報（任意） |
| 「保存」ボタン | 売上を登録 |

#### 削除確認ダイアログ
| 項目 | 説明 |
|------|------|
| 確認メッセージ | 「本当に削除しますか？」 |
| 「キャンセル」ボタン | 削除をやめる |
| 「削除」ボタン | 店舗を削除 |

### 操作の流れ

**予約リクエストを送る（このページから行います）：**
```
① 店舗カードの「詳細」ボタンをクリック
    ↓
② 店舗詳細モーダルが開く
    ↓
③ 「予約」タブをクリック
    ↓
④ 開始日・終了日を選択
    ↓
⑤ 担当者名を入力
    ↓
⑥ 「予約リクエストを送信」をクリック
    ↓
⑦ 予約代行者に通知される
    ↓
⑧ 承認されるとカレンダーにイベントが追加される
```

**売上を追加する：**
```
① 店舗カードの「粗利を追加」ボタンをクリック
    ↓
② ダイアログが開く
    ↓
③ 日付を選択
    ↓
④ 粗利金額を入力
    ↓
⑤ 買取品目数を入力
    ↓
⑥ 「保存」ボタンをクリック
    ↓
⑦ 「粗利を登録しました」と表示される
```

## 6. カレンダー・スケジュール

### 画面の説明
催事のスケジュールを管理する画面です。

### 画面の構成

#### タブ切り替え
| タブ名 | 説明 |
|--------|------|
| カレンダー | カレンダー形式で表示 |
| 一覧 | リスト形式で表示 |

#### カレンダー表示
| 項目 | 説明 |
|------|------|
| 月表示/週表示/日表示 | 表示期間を切り替え |
| イベントバー | 各催事を色付きバーで表示 |
| 「今日」ボタン | 今日の日付に移動 |
| 「<」「>」ボタン | 前後の期間に移動 |

#### イベントバーの色

| 色 | 状態 | 説明 |
|----|------|------|
| 青 | 予定 | これから開催 |
| 緑 | 実施中 | 現在開催中 |
| 紫 | 終了 | 開催終了 |
| グレー | キャンセル | 中止された催事 |
| 点線グレー | 要請中 | 承認待ちの予約リクエスト |

### ポップアップ

#### イベント詳細モーダル

**「基本情報」セクション**
| 項目 | 説明 |
|------|------|
| 店舗名 | 催事を行う店舗 |
| 開催期間 | 開始日〜終了日 |
| 担当者 | 担当者名 |
| ステータス | 予定/実施中/終了 |
| 概算コスト | 予定コスト |

**「粗利入力」セクション**
| 項目 | 説明 |
|------|------|
| 粗利金額 | 実際の売上を入力 |
| 買取品目数 | 買い取った品目数 |
| 「保存」ボタン | 粗利を登録 |

**ボタン**
| ボタン名 | 説明 |
|----------|------|
| Googleカレンダーに追加 | Googleカレンダーと同期 |
| ステータス変更 | 予定→実施中→終了 |
| 閉じる | モーダルを閉じる |

#### 予約リクエスト詳細モーダル

点線グレーの「要請中」イベントをクリックすると表示されます。

| 項目 | 説明 |
|------|------|
| 店舗名 | 申請した店舗 |
| 開催期間 | 希望日程 |
| 担当者 | 担当者名 |
| ステータス | 「承認待ち」 |
| 備考 | 申請時のメモ |

### 操作の流れ

**イベントの詳細を確認する：**
```
① カレンダーでイベントバーをクリック
    ↓
② イベント詳細モーダルが開く
    ↓
③ 情報を確認
    ↓
④ 「閉じる」で戻る
```

**売上（粗利）を入力する：**
```
① 終了したイベントをクリック
    ↓
② イベント詳細モーダルが開く
    ↓
③ 「粗利入力」セクションに移動
    ↓
④ 粗利金額を入力
    ↓
⑤ 買取品目数を入力
    ↓
⑥ 「保存」ボタンをクリック
    ↓
⑦ 「粗利を登録しました」と表示される
```

## 7. 売上分析

### 画面の説明
売上データをグラフや表で分析する画面です。

### 画面の構成

#### KPIカード（3つ）
| カード名 | 説明 |
|----------|------|
| 総粗利 | 全期間の粗利合計 |
| 総買取点数 | 全期間の買取品目数合計 |
| 平均粗利/催事 | 1催事あたりの平均粗利 |

#### タブ切り替え
| タブ名 | 説明 |
|--------|------|
| 催事売上 | 催事ごとの売上を表示 |
| 店舗売上 | 店舗ごとの売上を表示 |

#### フィルターエリア
| 項目 | 説明 |
|------|------|
| 検索 | 店舗名で検索 |
| 店舗で絞り込み | 特定の店舗のみ表示 |
| 表示期間 | 全期間/月別/週別 |

#### データテーブル
| カラム | 説明 |
|--------|------|
| 日付 | 売上の日付 |
| 店舗名 | 店舗の名前 |
| 粗利 | 売上金額 |
| 買取点数 | 品目数 |
| 操作 | 編集ボタン |

#### グラフ
| グラフ名 | 説明 |
|----------|------|
| 店舗別粗利ランキング | 粗利の高い店舗順 |
| 月別粗利推移 | 月ごとの粗利変化 |

### 操作の流れ

**売上を編集する：**
```
① テーブルの行にある「編集」ボタンをクリック
    ↓
② 入力欄が編集可能になる
    ↓
③ 金額や点数を修正
    ↓
④ 「保存」ボタンをクリック
    ↓
⑤ 「更新完了」と表示される
```

**期間で絞り込む：**
```
① 「表示期間」で「月別」を選択
    ↓
② データが月ごとにグループ化される
    ↓
③ 各月の合計が表示される
```

## 8. AI地域分析

### 画面の説明
AIを使って地域の買取ポテンシャルを分析する画面です。

### 画面の構成

#### 地域選択カード
| 項目 | 説明 |
|------|------|
| 都道府県 | 都道府県を選択 |
| 市区町村 | 市区町村を選択（任意） |
| 「AIで分析」ボタン | 分析を開始 |

### 分析結果表示（分析後に表示）

#### ヘッダー
| 項目 | 説明 |
|------|------|
| 地域名 | 分析した地域の名前 |
| 買取ポテンシャルバッジ | 高/中/低 で評価 |

#### 統計カード
| 項目 | 説明 |
|------|------|
| 人口 | 総人口 |
| 平均年齢 | 住民の平均年齢 |
| 平均年収 | 住民の平均年収 |
| 60歳以上比率 | 高齢者の割合 |
| 60歳以上女性比率 | 高齢女性の割合 |
| 住居エリア比率 | 住宅地の割合 |

#### AI分析コメント
| 項目 | 説明 |
|------|------|
| 分析コメント | AIによる地域の評価と提案 |
| ポテンシャル理由 | 評価の根拠 |
| データ出典 | 使用したデータの出所 |

### 買取ポテンシャルの見方

| バッジ | 意味 | 色 |
|--------|------|-----|
| 高 | 催事に最適な地域 | 緑 |
| 中 | 平均的な地域 | 黄 |
| 低 | 慎重に検討が必要 | 赤 |

### 操作の流れ

```
① 都道府県を選択（例：東京都）
    ↓
② 市区町村を選択（例：渋谷区）※任意
    ↓
③ 「AIで分析」ボタンをクリック
    ↓
④ 「分析中...」と表示される
    ↓
⑤ 分析結果が表示される
    ↓
⑥ 買取ポテンシャルと詳細を確認
```

## よくある質問（FAQ）

### Q: 予約リクエストとは何ですか？
A: 催事を開催したい店舗・日程を申請する機能です。「登録店舗」ページから行います。予約代行者が承認すると、カレンダーにイベントが自動登録されます。

### Q: ランク（S/A/B/C）とは何ですか？
A: AIが店舗を評価したランクです。60歳以上の女性人口、平均年収、住宅密度などを基に判定されます。Sが最も高評価です。

### Q: 「AI推定」バッジとは何ですか？
A: 公式統計データがない場合に、AIが推定した値であることを示します。

### Q: 予約代行者になるには？
A: 管理者が「組織設定」画面で予約代行者として登録する必要があります。

### Q: カレンダーの「要請中」とは？
A: まだ予約代行者に承認されていない予約リクエストです。承認されると通常のイベントになります。
